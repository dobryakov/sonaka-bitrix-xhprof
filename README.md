# Битрикс в Docker

Заготовка для локальной разработки: веб-контейнер (PHP 8.2 + Apache) и MariaDB 10.11. Файлы сайта монтируются с хоста; данные БД хранятся в именованном томе.

## Требования

- Docker и Docker Compose v2

## Быстрый старт

После команды `docker compose up -d` сервис **web** стартует только когда MariaDB пройдёт healthcheck (обычно несколько секунд).

### Уже есть готовый сайт

1. Скопируйте проект в каталог `www/` (в корне должны появиться `index.php`, `bitrix/`, `upload/` и т.д.).
2. Запустите `docker compose up -d --build` и откройте сайт в браузере.

### Установка с нуля (чистая установка)

Речь про продукт **[1С-Битрикс: Управление сайтом](https://www.1c-bitrix.ru/)** (коробочная CMS). Облачный **Битрикс24** так не ставится.

1. Поднимите контейнеры (каталог `www/` может быть пустым, кроме служебных файлов вроде `.gitkeep`):

   ```bash
   docker compose up -d --build
   ```

2. Скачайте официальный скрипт установки в корень сайта. Удобнее с хоста:

   ```bash
   curl -fsSL -o www/bitrixsetup.php "https://www.1c-bitrix.ru/download/scripts/bitrixsetup.php"
   ```

   Либо из контейнера (если на хосте нет `curl`):

   ```bash
   docker compose exec web bash -lc 'curl -fsSL -o /var/www/html/bitrixsetup.php "https://www.1c-bitrix.ru/download/scripts/bitrixsetup.php"'
   ```

3. Выдайте права на запись пользователю веб-сервера **`www-data`** — иначе мастер установки не создаст каталоги. Проще всего один раз отдать владение всему `www` из контейнера (для локальной разработки это обычно приемлемо):

   ```bash
   docker compose exec web bash -lc 'chown -R www-data:www-data /var/www/html'
   ```

   На Linux при монтировании с хоста новые файлы на диске могут снова оказаться «вашими» по владельцу; если установка жалуется на права — повторите `chown` или настройте UID/GID пользователя Apache под ваш `uid` (отдельная тонкость, для старта достаточно команды выше).

4. Откройте в браузере мастер установки:

   `http://localhost:8282/bitrixsetup.php`  
   (подставьте свой порт, если задали `WEB_PORT`.)

5. На шаге подключения к базе укажите параметры **из вашего `.env` или значений по умолчанию**:

   | Поле | Значение (по умолчанию в этом compose) |
   |------|----------------------------------------|
   | Сервер БД | `db` |
   | Пользователь | `bitrix` |
   | Пароль | `bitrixsecret` |
   | Имя базы | `bitrix` |

   База **`bitrix`** и пользователь уже создаются контейнером MariaDB при первом старте; отдельно создавать БД вручную не обязательно.

6. Дальше следуйте шагам мастера: редакция продукта, лицензионный ключ (для локальных тестов часто доступен демо-режим — смотрите подсказки на сайте 1С-Битрикс), скачивание дистрибутива и создание таблиц.

7. После успешной установки по документации Битрикс рекомендуется **удалить** `bitrixsetup.php` с сервера — в вашем случае удалите файл `www/bitrixsetup.php` на хосте.

Сайт после установки открывается по тому же адресу, что и мастер (корень сайта), например `http://localhost:8282/`.

## Сервисы

| Сервис | Описание |
|--------|----------|
| **web** | Сборка из `Dockerfile`: Apache, PHP с расширениями под Битрикс, том `./www` → `/var/www/html` |
| **db** | `mariadb:10.11`, utf8mb4, том `bitrix_db_data` для данных |

## Порты и доступ к БД

- **Веб:** хост-порт задаётся переменной `WEB_PORT` (по умолчанию `8282`).
- **MySQL с хоста:** порт `MYSQL_PORT` (по умолчанию `3306`) — удобно для клиентов вроде DBeaver.

**Из PHP внутри контейнера `web`** в настройках подключения указывайте:

- хост: **`db`**
- порт: **`3306`**
- имя БД / пользователь / пароль — как в переменных окружения ниже

Файл настроек Битрикс обычно: `www/bitrix/php_interface/dbconn.php` (или аналог в `.settings.php` в актуальных версиях).

## Переменные окружения

Задайте в оболочке или в файле `.env` в корне проекта:

| Переменная | По умолчанию | Назначение |
|------------|--------------|------------|
| `WEB_PORT` | `8282` | HTTP с хоста |
| `MYSQL_PORT` | `3306` | MySQL/MariaDB с хоста |
| `MYSQL_ROOT_PASSWORD` | `rootsecret` | root в MariaDB |
| `MYSQL_DATABASE` | `bitrix` | имя БД |
| `MYSQL_USER` | `bitrix` | пользователь приложения |
| `MYSQL_PASSWORD` | `bitrixsecret` | пароль пользователя |

Для продакшена замените пароли и не публикуйте `.env` в репозиторий.

## PHP и образ `web`

- Версия: **PHP 8.2**, **Apache** (Bookworm).
- Расширения: `mysqli`, `pdo_mysql`, `gd`, `intl`, `zip`, `soap`, `opcache`, `bcmath`, `calendar`, `exif`, `sockets`, `imagick` и стандартные зависимости сборки.
- Локаль в контейнере: **ru_RU.UTF-8**.
- Дополнительные директивы PHP: `docker/php-bitrix.ini` (лимиты памяти, `max_input_vars`, загрузки, OPcache).
- Виртуальный хост Apache: `docker/000-bitrix.conf` — в каталоге сайта включён **`AllowOverride All`** для `.htaccess` и ЧПУ.

При необходимости отредактируйте `docker/php-bitrix.ini` или конфиг Apache и пересоберите образ: `docker compose build web`.

## Структура репозитория

```
.
├── docker-compose.yml
├── Dockerfile
├── docker/
│   ├── 000-bitrix.conf    # виртуальный хост Apache
│   └── php-bitrix.ini      # доп. настройки PHP
└── www/                    # корень сайта (том для Битрикс)
```

## Полезные команды

```bash
docker compose logs -f web
docker compose exec web bash
docker compose exec db mariadb -ubitrix -p bitrix
```

Остановка с сохранением данных БД в томе:

```bash
docker compose down
```

Удаление контейнеров и **тома с БД** (данные будут потеряны):

```bash
docker compose down -v
```

## Ограничения заготовки

- Нет отдельных сервисов **Redis**, **Memcached**, **cron**, **push-сервера** — при необходимости их добавляют в `docker-compose.yml`.
- SSL на хосте обычно делают обратным прокси (nginx, traefik) снаружи compose.
